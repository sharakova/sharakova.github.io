ファッションコーディネートアプリ

300万DL突破 → 事業売却成功

iOS Android Rails PHP Exit

プロジェクト概要

女子大生をメインターゲットとしたファッションコーディネートアプリの企画から開発、グロース、そして事業売却まで、すべてのフェーズを担当しました。国内で300万ダウンロードを達成し、主要なファッションメディアとして成長させることに成功しました。

ブランドやファッション雑誌と提携し、独自コンテンツを生成。アジアを中心に海外展開も実施し、最終的に大手企業への事業売却（Exit）を実現しました。

担当領域

本プロジェクトでは、以下の幅広い領域を担当しました：

  • 事業企画: サービスコンセプトの立案、市場調査、競合分析
  • 開発マネジメント: ベトナムオフショア開発チームのブリッジエンジニア
  • iOS/Androidアプリ開発: ネイティブアプリの実装
  • バックエンド開発: Rails/PHPによるAPI設計・実装
  • グロース: マーケティング戦略、ユーザー獲得施策
  • パートナーシップ: ブランド・メーカー・雑誌社との提携交渉

主要機能

コーディネート投稿・共有機能

  • ユーザーが自身のファッションコーディネートを写真で投稿
  • 着用アイテムのタグ付け（ブランド、価格帯、購入場所など）
  • タイムライン形式で他ユーザーのコーディネートを閲覧
  • いいね・コメント・シェア機能

パーソナライズされたレコメンド

  • ユーザーの好みを学習し、最適なコーディネートを提案
  • 体型・年齢・予算に合わせたフィルタリング
  • 季節やトレンドを考慮したレコメンデーション

ショッピング機能

  • 投稿に含まれるアイテムから直接ECサイトへ遷移
  • ブランド公式サイトとの連携
  • アフィリエイト収益の仕組みを実装

オリジナルコンテンツ配信

  • ファッション雑誌・ブランドとのタイアップ記事
  • スタイリストによる着こなし提案
  • トレンド情報やコーディネート特集

技術スタック

モバイルアプリ

  • iOS: Swift、Objective-C（当時）
  • Android: Kotlin、Java
  • 画像処理: カスタムフィルター、リサイズ、最適化
  • プッシュ通知: Firebase Cloud Messaging

バックエンド（初期）

  • Ruby on Rails: RESTful API、管理画面
  • PostgreSQL: リレーショナルデータ管理
  • Redis: キャッシング、セッション管理
  • Elasticsearch: 高速な全文検索

バックエンド（スケール後）

  • PHP: 一部のマイクロサービス化
  • MySQL: スケーラビリティ向上のためのマイグレーション
  • Memcached: 追加のキャッシング層

インフラ

  • AWS EC2: アプリケーションサーバー（Auto Scaling）
  • AWS RDS: データベース（Multi-AZ）
  • AWS S3: 画像ストレージ
  • CloudFront: CDNによる画像配信
  • ELB: ロードバランシング

スケーリングの課題と解決策

課題1: 画像配信の最適化

課題: ユーザー数増加に伴い、画像配信のトラフィックが急増し、サーバー負荷が増大

解決策: CloudFront CDNを導入し、画像をエッジロケーションから配信。さらに、WebPフォーマットへの変換やレスポンシブ画像の実装により、データ転送量を50%削減しました。

課題2: データベースのボトルネック

課題: タイムライン取得やレコメンド機能で複雑なクエリが多く、レスポンス時間が低下

解決策: Redisによるキャッシング戦略を強化し、頻繁にアクセスされるデータをメモリ上に保持。また、データベースのインデックス最適化とクエリチューニングを実施し、レスポンスタイムを70%改善しました。

事業売却（Exit）までの道のり

成長フェーズでのKPI管理と、継続的な改善により、投資家・買収候補企業からの注目を集めました。最終的に、ファッション業界の大手企業から買収提案を受け、事業売却を実現しました。

この経験を通じて、プロダクト開発だけでなく、事業成長戦略、M&Aプロセスの実務まで、幅広い経験を積むことができました。
